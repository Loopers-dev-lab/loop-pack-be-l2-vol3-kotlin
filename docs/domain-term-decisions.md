# 도메인 용어 결정 사항

## OrderItem vs OrderProduct — 왜 OrderItem이 적절한가?

### 질문

핵심 도메인 용어에서 상품은 `Product`를 사용하는데, 주문 항목이 `OrderItem`인 것이 적절한가?
`OrderProduct`가 더 맞지 않은가?

### 결론: OrderItem이 적절하다

### 근거 1: OrderItem ≠ Product

요구사항에 명시되어 있다:

> 주문 정보에는 **당시 상품 정보의 스냅샷**이 저장되어야 한다 (상품 변경에 영향받지 않음)

`OrderItem`은 **주문 시점에 고정된 스냅샷**이다. `Product`와는 다른 개념이다:

| | Product | OrderItem |
|---|---|---|
| 성격 | 살아있는 카탈로그 데이터 | 주문 시점의 냉동된 스냅샷 |
| 가격 변경 | 반영됨 | 영향 없음 |
| 상품 삭제 | soft delete | 주문 이력에 그대로 남음 |
| 포함 정보 | 전체 상품 정보 | 상품명, 가격, 브랜드명 + **수량** |

### 근거 2: OrderProduct라는 이름의 문제

- `OrderProduct`는 "주문용 상품" 또는 "주문과 상품의 관계"처럼 들린다
- 실제로는 **수량(quantity)** 이라는 주문 고유 속성을 가지고 있어서, 단순한 Product 참조가 아니다
- 스냅샷 데이터까지 포함하므로 Product와는 완전히 다른 엔티티이다

### 근거 3: 업계 표준과의 일치

- `OrderItem` / `OrderLineItem` — Amazon, Shopify, 대부분의 이커머스에서 사용하는 표준 용어이다
- "주문 안에 포함된 하나의 항목"이라는 의미가 명확하다

### 정리

- **Product** = 브랜드에 속한 판매 가능한 상품 (카탈로그)
- **OrderItem** = 주문에 포함된 개별 상품 스냅샷과 수량 (주문 항목)

둘은 `productId`로 연결되지만, **서로 다른 생명주기를 가진 별개의 도메인 개념**이기 때문에 이름이 다른 것이 오히려 맞다.
