# 05. 설계 의문과 트레이드오프

## Q1. "왜 Brand와 Product는 ID 참조인가?"

| 방향 | 핵심 | 얻는 것 | 잃는 것 |
|------|------|---------|---------|
| A. ID 참조 | `Product.brandId: Long` | Aggregate 독립성, 트랜잭션 경계 분리 | 조회 시 조인 필요 |
| B. 객체 참조 | `Product.brand: Brand` | 직관적 탐색 | Aggregate 경계 모호, Lazy Loading 의존 |

**결정: A (ID 참조)** -- Brand와 Product는 생명주기가 독립적이고, 별도 트랜잭션으로 관리하는 것이 자연스럽다. 조회 시 필요한 브랜드 정보는 Facade 레이어에서 조합한다.

## Q2. "왜 주문에 상품 스냅샷을 저장하는가?"

주문 이후 상품 이름/가격이 변경되어도 주문 내역은 **주문 시점의 정보**를 보여야 한다. 이는 이커머스의 기본 원칙이며, OrderItem이 상품 정보의 복사본을 가지는 것이 정합성을 보장한다.

## Q3. "Like에 별도 VO가 필요한가?"

Like는 `memberId + productId` 조합이 본질이다. 별도 VO로 포장할 값이 없으므로 VO 없이 Long 타입으로 직접 참조한다. 이는 과도한 추상화를 피하는 결정이다.
