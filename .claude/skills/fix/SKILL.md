name: fix
description:
  결함 수정 TDD. 버그를 재현하는 테스트를 먼저 작성하고, 최소한의 수정으로 해결한다.
  API 레벨 재현 → 단위 테스트 재현 → 최소 수정 3단계로 진행한다.
---

결함 설명: $ARGUMENTS

## 절차

1. 결함 분석: 증상, 재현 조건, 기대 동작 파악
2. **Step 1** — API 레벨 실패 테스트 작성 (결함 재현 확인)
3. **Step 2** — 최소 재현 단위 테스트 작성 (근본 원인 좁히기)
4. **Step 3** — 두 테스트를 모두 통과시키는 최소한의 수정
5. 전체 테스트로 사이드 이펙트 확인: `./gradlew test`
6. 결과 보고: 원인 분석, 재현 테스트, 수정 내용, 전체 테스트 결과

## 규칙

- 반드시 실패 테스트 먼저 (테스트 없이 수정 금지)
- 수정은 결함 해결에 필요한 최소한의 범위
