  코드는 수정하지 않고, 불일치 항목만 리포트한다.
---

## 절차

1. **대상 문서 수집**: `docs/` 하위의 설계/요구사항 문서를 Glob으로 탐색한다
   - `docs/design/*.md`
   - `docs/requirements/*.md`
   - 사용자가 특정 문서를 지정하면 해당 문서만 대상으로 한다

2. **레이어 CLAUDE.md 수집**: 프로젝트의 모든 CLAUDE.md를 Glob(`**/CLAUDE.md`)으로 탐색하여 아키텍처 규칙 목록을 확보한다

3. **문서별 검증**: 각 문서에서 코드 참조(클래스명, 엔드포인트, 패턴, 규칙)를 식별하고, Grep/Read로 실제 코드와 대조한다

4. **보고 포맷**: 아래 형식으로 결과를 출력한다

```
## [문서명]

### 검증 통과
- ✅ [항목]: 코드에서 확인됨 (파일:라인)

### 불일치
- ❌ [항목]: 문서에는 X라고 되어 있으나, 코드에서는 Y로 구현됨 (파일:라인)

### 누락
- ⚠️ [항목]: 문서에 명시되어 있으나 코드에서 찾을 수 없음

### 문서 미반영
- 📝 [항목]: 코드에 존재하나 문서에 기술되지 않음
```

5. **요약 테이블**: 마지막에 전체 집계를 출력한다

```
| 문서 | ✅ 통과 | ❌ 불일치 | ⚠️ 누락 | 📝 미반영 |
```

## 규칙

- 코드를 **절대 수정하지 않는다** — 보고만 한다
- 불일치 항목에는 반드시 파일 경로와 라인 번호를 포함한다
- 판단이 모호한 항목은 "확인 필요"로 표시하고 개발자에게 넘긴다
