---
name: handoff
description:
  세션 종료 전 핸드오프 노트를 작성한다.
  다음 세션에서 컨텍스트 손실 없이 작업을 이어갈 수 있도록 한다.
---

## 절차

1. **현재 상태 수집**:
   - `git status`로 작업 중인 변경 파일 확인
   - `git log --oneline -5`로 최근 커밋 확인
   - `./gradlew test` 결과 (통과 여부, 테스트 수)

2. **핸드오프 노트 작성**: `.claude/handoff.md`에 아래 형식으로 작성한다

```markdown
# 세션 핸드오프

## 날짜
YYYY-MM-DD

## 완료된 작업
- [항목 1]
- [항목 2]

## 미완료 작업
- [ ] [항목]: 현재 상태 설명
- [ ] [항목]: 막힌 부분 설명

## 결정 사항
- [결정 1]: 이유
- [결정 2]: 이유

## 빌드/테스트 상태
- 빌드: 통과/실패
- 테스트: N개 통과 / N개 실패
- 린트: 통과/실패

## 다음 세션 시작 시 할 일
1. [첫 번째 할 일]
2. [두 번째 할 일]

## 주의사항
- [알아야 할 컨텍스트]
```

3. **개발자 확인**: 작성된 내용을 보여주고 수정/보완 여부를 확인한다

## 규칙

- 50줄 이내로 간결하게 유지한다
- 코드 스니펫은 포함하지 않는다 — 파일 경로와 라인 번호로 참조한다
- 다음 세션의 Claude가 이 파일만 읽고도 작업을 이어갈 수 있도록 핵심 컨텍스트를 빠짐없이 담는다
