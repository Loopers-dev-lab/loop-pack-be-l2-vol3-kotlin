PRD 또는 요구사항을 기반으로 plan.md를 작성하거나 업데이트한다.

요구사항/PRD: $ARGUMENTS

## 절차

1. `$ARGUMENTS`가 있으면 해당 요구사항 분석, 없으면 `PRD.md` 확인
2. 기존 `plan.md`가 있으면 진행 상황 파악 후 추가/수정 제안
3. 요구사항을 TDD로 구현할 수 있는 단위로 분해
4. 체크리스트 형태로 작성
5. 개발자 확인 후 저장

## plan.md 형식

```markdown
# 기능명

## 개요
요구사항 요약

## 구현 계획

### 1. 도메인 모델
- [ ] Entity 필수 필드 검증
- [ ] Entity 비즈니스 메서드

### 2. 저장소
- [ ] Repository 인터페이스
- [ ] RepositoryImpl 구현

### 3. 서비스
- [ ] 핵심 비즈니스 로직
- [ ] 에러 케이스 처리

### 4. API
- [ ] Controller 엔드포인트
- [ ] 요청/응답 DTO

### 5. E2E
- [ ] API 통합 테스트
```

## 규칙

- 각 항목은 하나의 테스트로 검증 가능한 크기
- 도메인 → 저장소 → 서비스 → API 순서 (의존성 방향)
- 항목은 구체적으로 (예: "로그인 ID가 빈 값이면 회원가입 실패")
- 개발자 확인 없이 임의 생성 금지
