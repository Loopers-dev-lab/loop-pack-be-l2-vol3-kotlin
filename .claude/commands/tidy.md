Tidy First: 동작 변경 없이 구조적 변경만 수행한다.

대상/방향: $ARGUMENTS

## 절차

1. 전체 테스트 통과 확인: `./gradlew test` (실패 시 중단)
2. 구조적 변경 수행 (한 번에 하나):
    - 이름 변경, 메서드 추출, 코드 이동
    - unused import 제거, 중복 제거
    - 가드 절 도입, 설명 변수 도입
3. 린트: `./gradlew ktlintFormat`
4. 전체 테스트 재확인: `./gradlew test` (실패 시 되돌리기)
5. 결과 보고

## 규칙

- **동작 변경 절대 금지**
- 한 번에 하나의 구조적 변경만
- 행위적 변경이 필요하면 별도 작업 제안
