# Claude Code 커스텀 커맨드

TDD (Red → Green → Refactor) + Tidy First + 설계 워크플로우에 맞춘 슬래시 커맨드 모음.

## 커맨드 목록

### TDD 사이클

| 커맨드         | 설명                   | 인자         |
|-------------|----------------------|------------|
| `/go`       | plan.md 기반 TDD 자동 수행 | 없음         |
| `/red`      | 실패 테스트 작성            | 요구사항 설명    |
| `/green`    | 테스트 통과 최소 구현         | 없음         |
| `/refactor` | 코드 품질 개선             | 대상/방향 (선택) |

### 검증 및 커밋

| 커맨드       | 설명               | 인자 |
|-----------|------------------|----|
| `/verify` | 린트 + 테스트 최종 검증   | 없음 |
| `/commit` | Tidy First 규칙 커밋 | 없음 |

### 계획 및 설계

| 커맨드              | 설명              | 인자            |
|------------------|-----------------|---------------|
| `/plan`          | plan.md 작성/업데이트 | 요구사항/PRD (선택) |
| `/design-review` | 설계 문서 전체 리뷰     | 없음            |

### 설계 산출물 (Week 2)

| 커맨드              | 설명           | 인자     |
|------------------|--------------|--------|
| `/sequence`      | 시퀀스 다이어그램 작성 | 대상 기능  |
| `/class-diagram` | 클래스 다이어그램 작성 | 대상 도메인 |
| `/erd`           | ERD 작성       | 대상 도메인 |

### 테스트 및 문서

| 커맨드     | 설명                | 인자          |
|---------|-------------------|-------------|
| `/e2e`  | E2E 테스트 작성        | 대상 API (선택) |
| `/http` | API HTTP 요청 파일 작성 | 대상 API (선택) |

### 결함 수정 및 정리

| 커맨드     | 설명              | 인자         |
|---------|-----------------|------------|
| `/fix`  | 버그 재현 테스트 → 수정  | 결함 설명      |
| `/tidy` | 동작 변경 없는 구조적 변경 | 대상/방향 (선택) |

## 일반적인 작업 흐름

### A. 설계 (Week 2)

```
(요구사항 분석은 /requirements-analysis 스킬 사용)
/sequence 주문 생성 흐름            (시퀀스 다이어그램)
/class-diagram 주문 도메인          (클래스 다이어그램)
/erd 주문 도메인                    (ERD)
/design-review                     (전체 설계 리뷰)
/commit                            (커밋)
```

### B. 새 기능 개발 (plan.md 기반)

```
/plan 요구사항 설명                 (plan.md 작성)
/go → /go → /go → ...             (plan.md 항목을 하나씩 진행)
/refactor                          (누적된 코드 정리)
/verify                            (최종 검증)
/commit                            (커밋)
/http                              (API HTTP 파일 작성)
```

### C. 단계별 수동 제어

```
/red 요구사항 설명                  (실패 테스트 작성)
/green                             (최소 구현)
/refactor                          (코드 정리)
/e2e                               (E2E 테스트 추가)
/verify                            (최종 검증)
/commit                            (커밋)
```

### D. 결함 수정

```
/fix 결함 설명                      (재현 테스트 + 수정)
/verify                            (최종 검증)
/commit                            (커밋)
```
