커밋 전 최종 검증: 린트 + 전체 테스트를 실행하고 결과를 보고한다.

## 절차

1. 린트 체크: `./gradlew ktlintCheck`
2. 전체 테스트: `./gradlew test`
3. 변경 파일 확인: `git status`
4. 결과를 아래 형식으로 보고:

```
## 검증 결과

### 린트 (ktlintCheck)
- 상태: 통과 / 실패
- 실패 시: 파일명, 위치, 위반 규칙

### 테스트
- 상태: 통과 / 실패
- 전체: N개 / 성공: N개 / 실패: N개

### 변경 파일
- 새 파일: N개 / 수정 파일: N개 / 삭제 파일: N개

### 판정
- 커밋 가능 → `/commit`으로 진행하세요
- 수정 필요 → 아래 항목을 수정하세요
```

## 규칙

- 린트 실패 시 `./gradlew ktlintFormat` 자동 수정 가능함을 안내
- 린트 + 테스트 모두 통과해야 "커밋 가능"
