# Git 커밋 실행 규칙

## 핵심 원칙

- 커밋을 요청받으면 **즉시 실행**한다. 불필요한 분석 루프(diff 반복 확인, 전략 재계획)에 빠지지 않는다
- 커밋 계획 파일이 제공되면 자체 판단 없이 해당 계획을 그대로 따른다
- 구조적 변경과 행위적 변경은 절대 같은 커밋에 섞지 않는다 (Tidy First)

## 로컬 전용 파일 보호

아래 패턴의 파일은 staging 전 반드시 개발자에게 확인한다:

- `*-plan.md`, `*-review.md` (작업 계획/리뷰 문서)
- `.omc/` 하위 파일
- 개인 메모성 파일

## 커밋 메시지

- 한국어로 작성
- 접두사: `feat:` | `refactor:` | `fix:` | `test:` | `docs:` | `chore:`
- 상세 절차는 `/commit` 스킬 참고
